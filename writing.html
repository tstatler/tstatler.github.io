The following lists a few writing samples from my work at Apcera (since acquired by Ericsson) who offered a multi-cloud, turn-key Container-as-a-Service (CaaS) platform.

    API Documentation

    I created all API documentation including reference guides, user guides, and sample code including:

    • Apcera Platform API Reference — API reference for REST managing a CaaS. All API's endpoints and resource types were described in a Swagger specification (JSON file), and HTML output was generated from via a custom build script that I developed.
    • Platform API Recipes — Companion guide to the Platform API reference (above) that provides recipes for specific API tasks, as well as the equivalent CLI commands. I tested all recipes using command line tools and wrote and edited all accompanying content.
    • Service Gateway API Overview — An overview and examples of the API used to connect containers (jobs) to external services, like MySQL.

    User Guides

    • Virtual Networks — Explains how to create "virtual" networks that allowed containers running on different hosts in different networks to communicate as if they were on the same network.
    • Configuring Sticky HTTP Sessions — Explains how to use session cookies to have client requests routed to the same backend container instance rather than use the default load balancing ("round robin").
    • Common Web Console Tasks — Provides instructions for performing common management tasks on your Apcera cluster using a web console.
