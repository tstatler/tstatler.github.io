API Reference and User Guides

I researched and wrote all API documentation for the Apcera container management system, including API reference, API user guides, sample code and sample apps, including:

    • Apcera Platform API Reference — API reference for managing an Apcera container cluster. All API endpoints and resource types were described in a Swagger specification ( JSON file ) and the HTML output was generated from a custom Ruby build script that I created.
    • Platform API Recipes — Companion guide to the Platform API reference (above) that provides recipes for specific API tasks, as well as the equivalent CLI commands. I tested all recipes using curl and PostMan and wrote all accompanying content.
    • Events System API — Explains use cases and provides working examples of the Events System API provided in the Apcera container management platform, including a quick feature demonstration using the Apcera CLI tool, as well a complete code example that demonstrates how to subscribe to events via WebSockets.

Developer Guides and Tutorials

    • Creating Your First Titanium App — Demonstrates how to create a multi-platform application for iOS and Android using the Titanium SDK. I posted the completed source files for the tutorial to Github.
    • Common Alloy Tasks — Explains how to use the Titanium Studio authoring tool to perform common setup and development tasks when working with the Alloy framework and the Titanium SDK to create mobile applications.
    • Virtual Networks — Explains how to create networks that allow Apcera container instances running on different hosts and networks to communicate via IP or named addresses.
    • Configuring Sticky HTTP Sessions — Explains how to use session cookies with the Apcera container management platform to have client requests routed to the same backend container instance, rather than use the default "round robin" load balancing algorithm.
    • Common Web Console Tasks — Provides instructions for performing common management tasks on an Apcera server cluster via a web console.
